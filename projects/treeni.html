Capstone Project · Jan 2026 – Present

Treeni – ESG Carbon Emissions Forecasting

A production-grade multi-series time-series forecasting system predicting Scope 1 and 2 carbon emissions across 30+ facilities for a multinational technology company, built entirely from scratch using Python.

Overview

The Problem

Organizations maintain historical ESG records but often lack reliable methods to project future values for planning, target setting, and sustainability reporting. In many real-world ESG scenarios, independent variables are not available, external drivers are inconsistent, and forecasting must be based strictly on historical time-series patterns.

This project builds a pure time-series forecasting system that predicts future values of ESG metrics using only historical observations across multiple scopes, cities, and sites independently.

Methodology

How It Was Built

The full pipeline runs in 5 phases, processing raw ESG data all the way through to Power BI-ready CSV outputs:

1. Data Preparation
2. Forecasting Models
3. Model Evaluation
4. Forecast Generation
5. Dashboard Visualization

Data was cleaned and structured using Pandas and NumPy, handling missing timestamps via forward fill and interpolation, detecting outliers with rolling mean and IQR methods, and applying KNN imputation for gaps. Each metric, scope, city, and site combination was treated as an independent time series.

Seven forecasting models were compared per series: Naive, ARIMA, SARIMAX, Holt's Linear Trend, Holt-Winters, Simple Exponential Smoothing, and Facebook Prophet. The best model was selected per series using MAE, RMSE, and MAPE with a chronological train/validation split.

Output

Results and Deliverables

The pipeline exports 12 Power BI-ready CSV files across three granularity levels (state, city, site), including model performance comparisons, backtest results, best-model forecasts, and all-model forecasts with confidence intervals.

An interactive Power BI dashboard was built allowing stakeholders to filter by metric, scope, city, and site and compare historical versus forecasted values with confidence bands.

The underlying dataset is confidential corporate data and cannot be shared publicly. The forecasting code is available on GitHub with placeholder data references. Full results and methodology are available upon request during interviews.
